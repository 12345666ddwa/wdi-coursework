知识点3：MICE多重插补的原理与算法

MICE（Multiple Imputation by Chained Equations，链式方程多重插补）是一种基于迭代回归的缺失值填补方法，不假设数据服从多元正态分布，适用于混合类型变量（连续+分类）。

核心算法逻辑

MICE的核心思想是：每个缺失变量都用其他变量作为预测变量，通过迭代回归逐步逼近稳定填补值。具体步骤：

1. 初始化：对所有缺失值用简单方法（如均值）填充
2. 迭代轮次 t = 1, 2, ..., M（通常M=10-20）：
  a) 对每个有缺失的变量Xⱼ：
     - 用Xⱼ的观测值(observed)对其他所有变量做回归
     - 从回归结果中提取参数 + 随机噪声
     - 用该模型预测Xⱼ的缺失值，并填入
     - 立即用新值更新数据集，供下一个变量使用
  b) 当一个轮次内所有变量都更新完毕，进入下一轮
3. 收敛后，保存当前完整数据集作为一份插补副本
4. 重复步骤2-3共m次（通常m=5-10），产生m份完整数据集
5. 对每份数据集分别分析，用Rubin规则合并结果

在WDI数据中的落地

本作业用到了5个指标（基尼系数、入学率、人均GDP、总人口、预期寿命），彼此的缺失模式不同。MICE利用指标间的相关关系——例如，基尼系数的缺失值可以由人均GDP和入学率来预测填补。最终保留全部2,821条观测，且补填后的基尼系数标准差（5.5）最接近删除完整行后的分布（7.3），远优于均值插补（4.4）。

优势 保留样本量、不扭曲方差、能处理混合变量类型、产生多个副本体现不确定性。

知识点4：局部异常因子（LOF）理论与应用

LOF（Local Outlier Factor，局部异常因子）是一种基于密度的异常检测算法。它不假定数据服从某种分布，而是通过比较一个点与其邻居的局部密度来判断异常——密度明显低于邻居的点即为异常。

如上所示，印度的人口z-score=10.27（远超3σ），摩纳哥的GDP z-score=5.80——这些在5维空间中被LOF标记为局部离群。注意：异常得分是负数，越小（越负）表示越异常。

理论原理

输入：数据集 X = {x₁, x₂, ..., xₙ}，参数 k（近邻数）

Step 1 · 计算k-距离：对每个点xᵢ，找到与其第k近的邻居的距离k-distance(xᵢ)

Step 2 · 可达距离：点xᵢ到xⱼ的可达距离 = max(k-distance(xⱼ), dist(xᵢ, xⱼ))
  ——确保邻居点的距离不小于其k-距离，稳定局部密度估计

Step 3 · 局部可达密度(lrd)：lrd(xᵢ) = k / Σ(可达距离之和)
  ——点周围越稠密，lrd越大

Step 4 · LOF得分：LOF(xᵢ) = (Σ lrd(邻居) / lrd(xᵢ)) / k
  ——如果点的密度远低于邻居密度，LOF > 1 → 可能异常
  ——LOF ≈ 1 → 正常；LOF > 1.5 → 可疑；LOF > 3 → 强烈异常信号

核心特点

  • 局部视角：同一个数据点放在稀疏区域是正常的，放在稠密区域可能就是异常——LOF能捕捉这种局部差异
  • 无分布假设：不要求数据服从正态分布，适合跨国宏观数据的复杂分布形态
  • 参数k：控制邻居范围，k太小会漏掉全局异常，k太大则失去局部性

在WDI跨国数据分析中的应用

本作业用LOF对5个核心指标做多变量异常检测（k=20，异常阈值=1.5）。发现以下典型异常类型：

  • 人口大国异常：中国、印度等因总人口远高于全球均值，在局部密度空间中成为离群点
  • 微型富裕国异常：摩纳哥、列支敦士登、卢森堡的人均GDP是周边国家的数倍，LOF得分极高
  • 数据错误型异常：某国家某年的基尼系数突然从35跳至98（录入错误），LOF成功捕获

关键区别 与IsolationForest相比，LOF更适合发现局部异常（如微型国在周边区域中的相对离群），而IsolationForest擅长发现全局孤立点。

Pandera 是一个Python库，允许你像定义数据库Schema一样定义Pandas DataFrame的列类型、取值范围、统计属性等规则，并在运行时自动校验。

注意：实际WDI数据中包含World、IDA total、IDA & IBRD total等非国家聚合实体。Pandera验证可以自动过滤这些行（因为它们的人口和人均GDP均为NaN），确保后续分析只针对主权国家。

验证逻辑

Pandera通过三个层次逐级检查：

# 第一层：类型检查（type checks）
schema = pa.DataFrameSchema({
  "gdp_per_capita": pa.Column(float,
    pa.Check.greater_than(0), # 必为正数
    pa.Check.less_than(200000)), # 不超过$200k
  "country_code": pa.Column(str,
    pa.Check.str_length(3, 3)), # 3位ISO代码
  "year": pa.Column(int,
    pa.Check.between(2000, 2022)), # 有效年份范围
  "gini": pa.Column(float,
    pa.Check(lambda s: s.between(20, 80)), # 自定义lambda
    nullable=True), # 允许缺失
})

# 一次调用即完成全部验证
validated_df = schema.validate(df) # 通过→返回DataFrame；失败→抛异常

验证层次

  • 类型层：检查列是否为目标数据类型（float/int/str）、是否允许缺失值、是否唯一
  • 值域层：通过内置检查（大于/小于/介于/正则/长度）或 lambda 自定义规则，确保每个值在合理范围内
  • 统计层：支持跨列检查（如`trade_pct_gdp + trade_pct_gdp_alt ≈ 100`）、分布检查（如`check_distinct_values_in_set`）、组内检查

在WDI数据中的落地

本作业定义了两套Schema：

  • 原始数据Schema：检查基尼系数(20-80)、入学率(0-100)、预期寿命(30-90)、人均GDP(>0)、人口(>0)、年份(2010-2022)
  • 清洗后Schema：额外检查无缺失值、无负数、国别代码为3位大写字母

特点 Pandera的验证是声明式的——你写规则，它帮你检查。一次定义，多次复用，适合数据管道的入口/出口检查点。

知识点6：数据管道验证要解决什么问题

数据管道验证（Data Pipeline Validation）是指在数据从采集到分析的全流程中，在每个关键节点设置质量检查，确保流入下一环节的数据是可靠的。

基于WDI实际数据的案例

以下为wdi_raw.parquet中的真实数据问题，直观展示管道验证的必要性：

# 实际数据中的"无声破坏"案例
案例1：South Sudan, 2016 → 人均GDP=NaN, 人口=NaN, 只有基尼系数=44.0和预期寿命=36.72
  → 如果跳过管道验证，计算均值时该行参与预期寿命统计但不参与GDP统计，均值被悄悄扭曲

案例2：IDA IBRD total, 2022 → 人均GDP=6036.81, 人口=NaN, 预期寿命=71.68
  → 这是区域汇总（非主权国家），如果不验证，人均GDP分组统计时它被算作一个国家

案例3：World, 2022-2018 → 人均GDP约$11,000-$12,000, 但人口总是NaN
  → "World"行虽然在API返回中有意义，但作为观测行参与分析是污染

要解决的四大问题

① 上游数据变更的无声破坏

WDI数据库每年更新，新版本的指标定义可能调整（如基尼系数的计算基期从2010变为2015）、单位可能变化（如GDP从当年美元变为不变价）、甚至某些指标被废弃。如果没有管道验证，代码可能在不报错的情况下算出错误结果。

② 数据漂移（Data Drift）

同一指标在不同年度的分布可能发生变化。例如COVID-19期间人均GDP大幅下降，如果管道中的统计假设（如"人均GDP标准差<5000"）来自疫情前的数据，验证规则会在漂移发生时自动告警，提醒分析师重新审视假设。

③ 代码修改的副作用

当你修改了缺失值填补策略（如从均值插补换成MICE），新的策略可能无意中改变了数据分布或引入不应出现的值。管道验证作为回归测试，确保修改前后的数据都符合Schema。

④ 可复现性与审计

在学术作业中，管道验证提供了数据血缘的可追溯性——每一批数据都经过同一套Schema的校验，结果可以被独立复现。这在APA报告规范和课程评分中至关重要。

# 典型数据管道的验证节点
def build_pipeline(df_raw):
  # 节点1：入口验证 —— 原始数据是否符合预期格式
  df = raw_schema.validate(df_raw)
  
  # 节点2：缺失值填补后验证
  df_imputed = mice_impute(df)
  df_imputed = imputed_schema.validate(df_imputed)
  
  # 节点3：异常值处理后验证
  df_clean = remove_anomalies(df_imputed)
  df_clean = clean_schema.validate(df_clean)
  
  # 节点4：出口验证 —— 最终数据质量报告
  return df_clean

核心价值 管道验证不是"找茬"，而是给数据质量上了保险——让"垃圾进垃圾出"变成"垃圾进，管道拦截"。在WDI作业中，如果基尼系数某年的值被Pandera拦截，你立刻知道是该手动检查还是换插补策略。

知识点2：分层抽样与加权统计量的关系

分层抽样：将总体按特征（如收入组）分成若干层，每层内独立抽样。

加权统计量：各层抽样比例≠总体比例，必须用权重校正，否则结果被过度代表的人群扭曲。

第四步：APA格式统计报告

研究问题：不同收入水平国家的经济发展指标差异及全球不平等趋势

方法：采用BCa Bootstrap（B=9,999）构建95%置信区间；Welch t检验比较6组收入组间差异，Cohen's d报告效应量，Bonferroni和BH-FDR双重校正控制多重比较误差；Mann-Kendall趋势检验评估2000-2022年全球不平等演变。

结果：四个收入组的GDP per capita 95%BCa置信区间完全不重叠，从低收入组（M=750.79, 95% BCa CI [648.63, 840.47]）到高收入组（M=40,483.99, 95% BCa CI [36,480.80, 46,001.59]），组间差异约55倍。所有6组GDP比较的Welch t检验均显著（pBH<0.001），效应量均为large（d=-3.22~-1.35）。预期寿命从低收入组（M=61.77岁）到高收入组（M=79.49岁）递增（d=-2.47, pBH<0.001）。Mann-Kendall趋势检验表明2000-2022年间全球国家间GDP的CV（S=-161, p<0.001）和Gini系数（S=-159, p<0.001）均显著下降，降幅分别为22.1%和13.2%，全球不平等呈收敛趋势。

讨论：高收入组的组内方差远大于其他三组（SD=22,205），说明高收入国家间的分化程度最高。基尼系数的组间差异仅在BH-FDR下显著（Bonferroni不显著），建议在探索性研究中优先使用FDR校正，以提高检测力。全球不平等的持续收敛趋势在COVID-19后未逆转。

研究问题：什么因素决定了各国经济增长率？初始经济水平较低的国家是否增长更快（条件收敛假说）？

模型公式

被解释变量：growth_rate = (GDPpc_2020 - GDPpc_2010) / GDPpc_2010 （10年平均增长率）

解释变量：

模型：growth_rate = β₀ + β₁×ln(GDP_2010) + β₂×edu + β₃×trade + β₄×life + β₅×inf + ε
